# Adding Images to GitHub

This folder contains screenshots and visualizations for the project README.

## Image Files Needed

Add the following screenshot files to this folder:

1. **prediction-interface.png** - Main prediction interface showing the web app
2. **model-information.png** - Model architecture and performance metrics
3. **data-overview.png** - Dataset statistics and call type distribution chart
4. **sample-data.png** - Sample data table and detailed statistics

## How to Add Images

### Step 1: Take Screenshots
Capture screenshots of your application:
- Main prediction page
- Model information page
- Data overview page
- Sample data page

### Step 2: Save Images
1. Save screenshots as PNG files (recommended for GitHub)
2. Name them exactly as listed above
3. Place them in this `images/` folder

### Step 3: Add to Git
```bash
git add images/
git commit -m "Add application screenshots"
git push origin main
```

### Step 4: Verify
After pushing, check your GitHub repository to ensure images display correctly in the README.

## Image Format Recommendations

- **Format**: PNG (best for screenshots) or JPG
- **Size**: Keep under 1MB per image for faster loading
- **Dimensions**: 1200-1600px width is ideal for GitHub README
- **Naming**: Use lowercase with hyphens (e.g., `prediction-interface.png`)
